import
 - 특정 파일을 불러오는 것
 - 리액트로 만든 프로젝트의 자바스크립트 파일에서는 import를 이용해 다른 파일을 불러와 사용한다.
    -> Node.js지원하는 기능 -> Node.js에서는 require이라는 구문으로 불러옴

 - 브라우저에서 사용하기 위해 번들러(bundler)를 사용해 모듈을 합쳐서 하나의 파일을 생성
    -> 대표적인 번들러로는 웹팩, parcel, browserify가 있다.
    -> 현재는 웹팩을 많이씀 -> 편의성과 확장성이 뛰어남 -> SVG나 CSS파일도 불러 올 수 있다.
    -> 웹팩의 로더(loader)의 기능임
        ->css-loader / file-loader / babel-loader가 있다.
 - 웹팩의 로더는 create-react-app을 통해 대신 해주기 때문에 별도 설정을 해줄 필요는 없다.

JSX
  - 자바스크립트의 확장 문법, XML과 유사함
  - React.createElement함수를 사용하는 불편함을 없애줌
 장점
    - 보기 쉽고 익숙함 -> HTML코드를 보는 것과 유사함
    - 활용도가 높다. -> HTML태그들을 사용할 수 있을 뿐 아니라
                         각종 컴포넌트도 JSX안에서 작성 가능함
 규칙
    - 컴포넌트에 여러 요소가 있따면 반드시 부모 요소 하나로 감싸야 한다.
        -> Virtual DOM에서 컴포넌트 변화 감지를 효율적으로 비교할 수 있도록
         하나의 DOM트리 구조로 이루어져야함
    - JSX내부에서 자바스크립트 표현식을 사용하기 위해서는 {}로 감싸야함
    - if문은 사용할 수 없다. -> 조건부연산자를 이용 / 외부에 값을 설정
    - undefined를 랜더링 하면 안된다.
    - 인라인 스타일링 시 - 문자를 없애고 카멜표기법으로 작성해야 한다.
    - class를 사용할 때는 className으로 사용해야한다.
    - 태그들은 꼭 닫아주어야 한다 -> self-closing 태그를 통해 닫는다.
