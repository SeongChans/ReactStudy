라우팅이란 

    - 사용자가 요청한 URL에 따라 알맞은 페이지를 보여주는 것
    - 여러 페이지로 구성된 웹 어플리케이션을 만들 때 페이지 별로 컴포넌트들을
        분리해서 프로젝트를 관리하기 위해 필요한 것
종류
    - 리액트 라우터 (React Router) - 리액트의 라우팅 관련 라이브러리들 중에서 가장 오래되고 가장 많이 사용됨
        - 컴포넌트 기반으로 라우팅 시스템을 설정 할 수 있다
    - Next.js - 프레임워크중 하나 
        - 리액트 프로젝트 설정 / 라우팅 시스템 / 최적화 / 다국어 시스템 지원 / 서버 사이드 렌더링 등의 
        다양한 기능을 제공
    - 기타 - react-location, rakkas등이 있다.

싱글페이지 어플리케이션이란

    - 하나의 페이지로 이루어진 어플리케이션
    - HTML은 한번만 받아와서 실행시킨 후, 이후에는 필요한 데이터만 받아 화면에 업데이트 하는 것
    - 정적인 페이지에서는 HTML을 교환하는 방식도 괜찮지만 / 사용자 인터랙션이 많고 다양한 정보를 제공하는 
        모던 웹 어플리케이션은 적합하지 않다.
    - 리액트 같은 라이브러리를 통해 뷰 렌더링을 사용자의 브라우저가 담당토록 하고
        사용자와의 인터랙션이 발생하면 필요한 부분만 자바스크립트를 이용해 업데이트하는 방식을 사용함
        - 새운 데이터만 불러와 어플리케이션에서 사용할 수 있게됨

URL파라매터와 쿼리스트링

    - URL파라매터 - /porfile/velopert
    - 쿼리스트링 - /articles?page=1&keyword=react
    - URL파라캐터는 주소의 경로에 유동적인 값을 넣는 형태
        - 주로 ID 또는 이름을 사용하여 특정 데이터를 조회할 때 사용
    - 쿼리 스트링은 주소의 뒷부분에 ? 문자열 이후에 key=value로 값을 정의하며 &로 구분하는 형태
        - 키워드 검색, 페이지네이션, 정렬 방식 등 데이터 조회에 필요한 옵션을 전달할 때 사용
